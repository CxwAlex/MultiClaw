# MultiClaw 多实例架构修复方案

## 一、问题诊断

### 1.1 目录结构问题

**现状**：
```
~/.multiclaw/
├── config.toml
├── workspace/              # ❌ 董事长文件被放在这里
│   ├── IDENTITY.md
│   ├── SOUL.md
│   └── ...
└── instances/
```

**期望**：
```
~/.multiclaw/
├── config.toml
├── IDENTITY.md             # ✅ 董事长文件应该在第一层
├── SOUL.md
├── AGENTS.md
├── USER.md
├── MEMORY.md
└── instances/
```

**根本原因**：
- `scaffold_workspace()` 在 `onboard/wizard.rs` 中将文件写入 `workspace_dir`
- `workspace_dir` 默认是 `~/.multiclaw/workspace`，而不是 `~/.multiclaw`
- `ensure_chairman_files()` 在 daemon 启动时写入 `workspace_dir`

### 1.2 多实例支持问题

**现状**：
- 只支持单个董事长实例
- 公司实例创建后进程管理混乱

**期望**：
- 支持多个董事长实例（通过 `--config-dir` 参数）
- 每个董事长实例有独立的目录空间
- 公司实例进程独立运行，状态持久化

### 1.3 初始化流程问题

**现状**：
- `multiclaw onboard` → 创建 workspace 目录下的文件
- `multiclaw daemon` → 调用 `ensure_chairman_files()`，但路径可能不对
- `multiclaw service restart` → 不触发初始化逻辑

**期望**：
- 所有启动命令（daemon、service start/restart）都应确保董事长文件存在
- 初始化逻辑统一，不依赖特定命令

### 1.4 技能注入时机问题

**现状**：
```rust
// src/agent/loop_.rs:1493
let create_company_tool = Box::new(tools::CreateCompanyTool::with_config(...));
tools_registry.push(create_company_tool);
tracing::info!("Chairman tool 'create_company' registered");
```
- 工具在对话开始时才注册
- 不是董事长身份绑定的

**期望**：
- 董事长技能在实例初始化时就注入
- 技能与身份绑定

### 1.5 实例进程管理问题

**现状**：
- `CreateCompanyTool` 使用静态变量 `NEXT_PORT` 分配端口
- 进程状态存储在内存中，重启后丢失
- 没有健康检查机制

**期望**：
- 端口分配检查可用性
- 状态持久化到文件
- 完善的健康检查和恢复机制

```rust
pub struct InstanceInitializer {

    /// 实例类型
    instance_type: InstanceType,
}

#[derive(Clone, Copy, PartialEq)]
pub enum InstanceType {
    /// 董事长实例（默认实例）
    Chairman,
    /// 公司实例（CEO）
    Company,
    /// 团队实例
    Team,

impl InstanceInitializer {
    /// 确保实例文件存在
    pub async fn ensure_initialized(&self) -> Result<InitializationResult> {
        match self.instance_type {
            InstanceType::Chairman => self.ensure_chairman_files().await,
            InstanceType::Company => self.ensure_company_files().await,
            InstanceType::Team => self.ensure_team_files().await,
        }
    }

    /// 确保董事长文件存在
    async fn ensure_chairman_files(&self) -> Result<InitializationResult> {
        let required_files = [
            "IDENTITY.md", "SOUL.md", "AGENTS.md", 
            "USER.md", "MEMORY.md", "TOOLS.md"
        ];
        
        let mut created = 0;
        for filename in &required_files {
            let path = self.root_dir.join(filename);
            if !path.exists() {
                let content = get_default_chairman_content(filename);
                fs::write(&path, content).await?;
                created += 1;
            }
        }
        
        // 确保子目录存在
        for dir in &["sessions", "memory", "state", "cron", "skills", "instances"] {
            fs::create_dir_all(self.root_dir.join(dir)).await?;
        }
        
        Ok(InitializationResult { created, skipped: 6 - created })
    }
}
```

### 2.4 实例进程管理重构

#### 2.4.1 端口分配改进

```rust
// src/instance/port_manager.rs (新文件)

use std::sync::atomic::{AtomicU16, Ordering};
use std::collections::HashSet;

pub struct PortManager {
    next_port: AtomicU16,
    allocated_ports: RwLock<HashSet<u16>>,
    base_port: u16,
}

impl PortManager {
    pub fn new() -> Self {
        Self {
            next_port: AtomicU16::new(8001),
            allocated_ports: RwLock::new(HashSet::new()),
            base_port: 8001,
        }
    }
    
    /// 分配一个可用端口
    pub async fn allocate(&self) -> Result<u16> {
        let max_attempts = 100;
        for _ in 0..max_attempts {
            let port = self.next_port.fetch_add(1, Ordering::SeqCst);
            
            // 检查端口是否已被分配
            {
                let allocated = self.allocated_ports.read().await;
                if allocated.contains(&port) {
                    continue;
                }
            }
            
            // 检查端口是否可用
            if self.is_port_available(port).await? {
                self.allocated_ports.write().await.insert(port);
                return Ok(port);
            }
        }
        
        Err(anyhow!("No available ports after {} attempts", max_attempts))
    }
    
    /// 释放端口
    pub async fn release(&self, port: u16) {
        self.allocated_ports.write().await.remove(&port);
    }
    
    /// 从持久化状态恢复已分配端口
    pub async fn restore_from_state(&self, ports: Vec<u16>) {
        let mut allocated = self.allocated_ports.write().await;
        for port in ports {
            allocated.insert(port);
            // 更新 next_port 确保不会分配已使用的端口
            if port >= self.next_port.load(Ordering::SeqCst) {
                self.next_port.store(port + 1, Ordering::SeqCst);
            }
        }
    }
    
    async fn is_port_available(&self, port: u16) -> Result<bool> {
        let addr: std::net::SocketAddr = format!("127.0.0.1:{}", port).parse()?;
        match tokio::net::TcpListener::bind(addr).await {
            Ok(listener) => {
                drop(listener);
                Ok(true)
            }
            Err(e) if e.kind() == std::io::ErrorKind::AddrInUse => Ok(false),
            Err(e) => Err(e.into()),
        }
    }
}
```

#### 2.4.2 实例状态持久化

```rust
// src/instance/state.rs (新文件)

use serde::{Deserialize, Serialize};
use std::path::Path;
use tokio::fs;

/// 实例状态（持久化）
#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct InstanceState {
    pub id: String,
    pub name: String,
    pub instance_type: String,
    pub port: u16,
    pub pid: Option<u32>,
    pub data_dir: String,
    pub config_file: String,
    pub status: InstanceStatus,
    pub created_at: String,
    pub updated_at: String,
}

#[derive(Clone, Copy, Debug, Serialize, Deserialize)]
pub enum InstanceStatus {
    Creating,
    Starting,
    Running,
    Stopped,
    Crashed,
    Unknown,
}

/// 实例状态管理器
pub struct InstanceStateManager {
    state_file: PathBuf,
}

impl InstanceStateManager {
    pub fn new(root_dir: &Path) -> Self {
        Self {
            state_file: root_dir.join("state").join("instances.json"),
        }
    }
    
    /// 加载所有实例状态
    pub async fn load_all(&self) -> Result<Vec<InstanceState>> {
        if !self.state_file.exists() {
            return Ok(Vec::new());
        }
        
        let content = fs::read_to_string(&self.state_file).await?;
        let states: Vec<InstanceState> = serde_json::from_str(&content)?;
        Ok(states)
    }
    
    /// 保存所有实例状态
    pub async fn save_all(&self, states: &[InstanceState]) -> Result<()> {
        if let Some(parent) = self.state_file.parent() {
            fs::create_dir_all(parent).await?;
        }
        
        let content = serde_json::to_string_pretty(states)?;
        fs::write(&self.state_file, content).await?;
        Ok(())
    }
    
    /// 添加或更新实例状态
    pub async fn upsert(&self, state: &InstanceState) -> Result<()> {
        let mut states = self.load_all().await?;
        
        // 查找并更新，或添加新状态
        if let Some(existing) = states.iter_mut().find(|s| s.id == state.id) {
            *existing = state.clone();
        } else {
            states.push(state.clone());
        }
        
        self.save_all(&states).await
    }
    
    /// 删除实例状态
    pub async fn remove(&self, instance_id: &str) -> Result<()> {
        let mut states = self.load_all().await?;
        states.retain(|s| s.id != instance_id);
        self.save_all(&states).await
    }
}
```

#### 2.4.3 实例进程管理器

```rust
// src/instance/manager.rs (重构)

pub struct InstanceManager {
    port_manager: Arc<PortManager>,
    state_manager: InstanceStateManager,
    processes: RwLock<HashMap<String, ProcessInfo>>,
}

impl InstanceManager {
    /// 启动时恢复实例状态
    pub async fn restore_instances(&self) -> Result<Vec<String>> {
        let states = self.state_manager.load_all().await?;
        let mut restored = Vec::new();
        
        for state in states {
            // 检查进程是否还在运行
            if let Some(pid) = state.pid {
                if self.is_process_running(pid).await {
                    // 进程还在运行，恢复状态
                    self.port_manager.restore_from_state(vec![state.port]).await;
                    restored.push(state.id.clone());
                    tracing::info!("Restored instance: {} (port: {}, pid: {})", 
                        state.id, state.port, pid);
                } else {
                    // 进程已退出，更新状态
                    let mut state = state;
                    state.status = InstanceStatus::Crashed;
                    state.pid = None;
                    self.state_manager.upsert(&state).await?;
                }
            }
        }
        
        Ok(restored)
    }
    
    /// 创建并启动新实例
    pub async fn create_instance(&self, request: CreateInstanceRequest) -> Result<InstanceState> {
        // 1. 分配端口
        let port = self.port_manager.allocate().await?;
        
        // 2. 创建实例目录和配置
        let instance_id = self.create_instance_files(&request, port).await?;
        
        // 3. 启动进程
        let pid = self.start_process(&instance_id, port).await?;
        
        // 4. 保存状态
        let state = InstanceState {
            id: instance_id,
            name: request.name,
            instance_type: format!("{:?}", request.instance_type),
            port,
            pid: Some(pid),
            data_dir: request.data_dir,
            config_file: request.config_file,
            status: InstanceStatus::Running,
            created_at: chrono::Utc::now().to_rfc3339(),
            updated_at: chrono::Utc::now().to_rfc3339(),
        };
        
        self.state_manager.upsert(&state).await?;
        
        Ok(state)
    }
    
    /// 停止实例
    pub async fn stop_instance(&self, instance_id: &str) -> Result<()> {
        let states = self.state_manager.load_all().await?;
        let state = states.iter()
            .find(|s| s.id == instance_id)
            .ok_or_else(|| anyhow!("Instance not found: {}", instance_id))?;
        
        if let Some(pid) = state.pid {
            // 发送 SIGTERM
            #[cfg(unix)]
            {
                use nix::sys::signal::{kill, Signal};
                kill(Pid::from_raw(pid as i32), Signal::SIGTERM).ok();
            }
            
            // 等待进程退出
            tokio::time::sleep(Duration::from_secs(2)).await;
            
            // 如果还在运行，强制杀死
            if self.is_process_running(pid).await {
                #[cfg(unix)]
                {
                    kill(Pid::from_raw(pid as i32), Signal::SIGKILL).ok();
                }
            }
        }
        
        // 释放端口
        self.port_manager.release(state.port).await;
        
        // 更新状态
        let mut state = state.clone();
        state.status = InstanceStatus::Stopped;
        state.pid = None;
        self.state_manager.upsert(&state).await?;
        
        Ok(())
    }
}
```

### 2.5 多实例管理脚本

```bash
#!/bin/bash
# multiclaw-instance.sh - MultiClaw 多实例管理脚本

INSTANCES_DIR="$HOME/.multiclaw_instances"
DEFAULT_INSTANCE="$HOME/.multiclaw"

# 创建新实例
create_instance() {
    local name="$1"
    local instance_dir="$INSTANCES_DIR/$name"
    
    if [[ -d "$instance_dir" ]]; then
        echo "❌ 实例 '$name' 已存在"
        exit 1
    fi
    
    mkdir -p "$instance_dir"
    
    # 初始化实例
    multiclaw onboard --config-dir "$instance_dir"
    
    echo "✅ 实例 '$name' 创建成功"
    echo "   目录: $instance_dir"
    echo "   启动: multiclaw daemon --config-dir $instance_dir"
}

# 启动实例
start_instance() {
    local name="$1"
    local instance_dir=$(get_instance_dir "$name")
    
    if [[ -z "$instance_dir" ]]; then
        echo "❌ 实例 '$name' 不存在"
        exit 1
    fi
    
    multiclaw daemon --config-dir "$instance_dir"
}

# 列出所有实例
list_instances() {
    echo "📋 MultiClaw 实例列表:"
    echo ""
    
    # 默认实例
    if [[ -f "$DEFAULT_INSTANCE/config.toml" ]]; then
        echo "  [默认] ~/.multiclaw"
    fi
    
    # 其他实例
    if [[ -d "$INSTANCES_DIR" ]]; then
        for dir in "$INSTANCES_DIR"/*; do
            if [[ -d "$dir" ]]; then
                local name=$(basename "$dir")
                echo "  [$name] $dir"
            fi
        done
    fi
}

# 获取实例目录
get_instance_dir() {
    local name="$1"
    
    if [[ "$name" == "default" ]] || [[ -z "$name" ]]; then
        echo "$DEFAULT_INSTANCE"
        return
    fi
    
    local instance_dir="$INSTANCES_DIR/$name"
    if [[ -d "$instance_dir" ]]; then
        echo "$instance_dir"
    fi
}

# 主命令
case "$1" in
    create)
        create_instance "$2"
        ;;
    start)
        start_instance "$2"
        ;;
    stop)
        stop_instance "$2"
        ;;
    list)
        list_instances
        ;;
    *)
        echo "用法: $0 {create|start|stop|list} [name]"
        exit 1
        ;;
esac
```

## 五、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 目录结构变更导致旧数据不兼容 | 用户数据丢失 | 提供迁移脚本，自动移动文件 |
| 进程管理变更导致实例无法启动 | 服务中断 | 完善错误处理和日志 |
| 端口冲突 | 实例启动失败 | 端口可用性检查，自动重试 |
| 状态持久化失败 | 数据丢失 | 原子写入，备份机制 |
