# MultiClaw 多实例架构修复 - 实施进度

## 已完成的修复

### 1. 目录结构修复 ✅

**修改文件**：
- `src/onboard/wizard.rs` - `scaffold_workspace()` 函数
- `src/daemon/mod.rs` - `ensure_chairman_files()` 函数

**修改内容**：
- 董事长文件（IDENTITY.md, SOUL.md, AGENTS.md 等）现在写入 `config_dir`（如 `~/.multiclaw`）
- 数据目录（sessions, memory, state, cron, skills, instances）保持在 `workspace_dir` 下

**新的目录结构**：
```
~/.multiclaw/                           # 董事长实例根目录
├── config.toml                         # 全局配置
├── chairman_config.toml                # 董事长配置
├── IDENTITY.md                         # 董事长身份 ✅
├── SOUL.md                             # 董事长角色 ✅
├── AGENTS.md                           # 董事长操作指南 ✅
├── USER.md                             # 用户信息 ✅
├── MEMORY.md                           # 董事长记忆 ✅
├── TOOLS.md                            # 工具笔记 ✅
├── workspace/                          # 工作数据目录
│   ├── sessions/                       # 会话记录
│   ├── memory/                         # 记忆存储
│   ├── state/                          # 状态文件
│   ├── cron/                           # 定时任务
│   ├── skills/                         # 用户技能
│   └── instances/                      # 公司实例目录
└── instances/                          # 公司实例目录（新位置）
    └── {company_id}/                   # 公司实例
        ├── config.toml
        ├── IDENTITY.md                 # CEO 身份
        ├── SOUL.md                     # CEO 角色
        └── teams/                      # 团队目录
```

### 2. 初始化流程统一 ✅

**修改文件**：
- `src/daemon/mod.rs`

**修改内容**：
- `ensure_chairman_files()` 现在接收 `config_dir` 和 `workspace_dir` 两个参数
- 在 `daemon` 启动时正确计算 `config_dir = config.config_path.parent()`

### 3. 公司实例目录修复 ✅

**修改文件**：
- `src/tools/create_company.rs`

**修改内容**：
- `CreateCompanyTool` 现在使用 `config_dir.join("instances")` 作为实例目录
- 添加了 `config_dir` 字段和 `with_dirs()` 构造函数

### 4. 多实例管理脚本 ✅

**新建文件**：
- `scripts/multiclaw-instance.sh`

**功能**：
- 创建新实例：`./multiclaw-instance.sh create <name>`
- 启动实例：`./multiclaw-instance.sh start <name>`
- 停止实例：`./multiclaw-instance.sh stop <name>`
- 列出实例：`./multiclaw-instance.sh list`
- 查看状态：`./multiclaw-instance.sh status [name]`
- 删除实例：`./multiclaw-instance.sh remove <name>`
- 查看日志：`./multiclaw-instance.sh logs <name>`
- 编辑配置：`./multiclaw-instance.sh config <name>`

## 待完成的优化

### 5. 技能注入重构

**当前问题**：
- `CreateCompanyTool` 在 `agent/loop_.rs` 中注册（对话开始时）
- 应该在董事长实例初始化时就注入

**修改计划**：
1. 修改 `ChairmanAgent::initialize_with_config()`，在初始化时注册董事长技能
2. 移除 `loop_.rs` 中的工具注册代码

### 6. 实例进程管理改进

**当前问题**：
- 端口分配使用静态变量，不检查可用性
- 进程状态存储在内存中，重启后丢失
- 没有健康检查机制

**修改计划**：
1. 创建 `PortManager` 模块，检查端口可用性
2. 创建 `InstanceStateManager`，持久化状态到 JSON 文件
3. 完善 `InstanceManager` 的健康检查

### 手动测试步骤

1. **清理旧数据**：
```bash
rm -rf ~/.multiclaw
```

2. **运行 onboard**：
```bash
multiclaw onboard
```

3. **验证目录结构**：
```bash
ls -la ~/.multiclaw/
# 应该看到 IDENTITY.md, SOUL.md 等在第一层
ls -la ~/.multiclaw/workspace/
# 应该看到 sessions/, memory/ 等子目录
```

4. **启动 daemon**：
```bash
multiclaw daemon
```

5. **创建公司实例**：
```bash
multiclaw agent -m "帮我创建一个市场研究公司"
```

6. **验证公司实例**：
```bash
ls -la ~/.multiclaw/instances/
# 应该看到公司实例目录
```

## 使用指南

### 单个董事长实例（默认）

```bash
# 初始化
multiclaw onboard

# 启动
multiclaw daemon

# 对话
multiclaw agent -m "你好"
```

### 多个董事长实例

```bash
# 创建新实例
./scripts/multiclaw-instance.sh create project_a

# 启动实例
./scripts/multiclaw-instance.sh start project_a

# 或手动启动
multiclaw daemon --config-dir ~/.multiclaw_instances/project_a

# 列出所有实例
./scripts/multiclaw-instance.sh list
```

## 注意事项

1. **编译后测试**：修改后需要重新编译才能生效
   ```bash
   cargo build --release
   ```

2. **服务重启**：如果使用 systemd/launchd 服务，需要重启服务
   ```bash
   multiclaw service restart
   ```

3. **数据迁移**：如果之前有数据在 `~/.multiclaw/workspace/` 下，需要手动移动到正确位置
   ```bash
   mv ~/.multiclaw/workspace/IDENTITY.md ~/.multiclaw/
   mv ~/.multiclaw/workspace/SOUL.md ~/.multiclaw/
   # ... 其他文件
   ```
